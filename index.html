Práctica - Sistema de control de versiones

Usuario 1

El usuario1 clona el repositorio de GitHub recién creado y ejecuta el comando git flow init:

Después el usuario1 se mueve a la rama develop con el comando git checkout develop, crea la estructura del proyecto, añade el código necesario a index.html y crea app.js.

Por último, añade los cambios al área de preparación usando el comando git add ., crea un commit con git commit -m "mensaje" y sube los cambios al repositorio GitHub con git push origin develop.

Usuario 2

El usuario2 clona el repositorio de GitHub con los cambios hechos por el usuario1 y vuelve a ejecutar el comando git flow init.

Después ejecuta el comando git flow feature start contingutHTML, que creará una nueva rama llamada feature/contingutHTML.

A continuación, el usuario2 añade el contenido necesario al HTML y JS mediante git add y git commit -m "mensaje".

Para terminar la feature usa git flow feature finish contingutHTML, lo que fusiona los cambios en la rama develop.

Estos pasos se repiten para la feature: feature/atributsHTML

Por último subimos los cambios al repositorio GitHub con el comando git push origin develop
